```bash

git clone https://github.com/evdmatvey/demo

# Установите зависимости
npm install
```

### Краткое описание принятых решений

#### Универсальность компонента

Компонент `NewsList` принимает параметры `variant`, `type` и `newsPerPage`, что позволяет использовать его как для новостей компании, так и для бизнес-новостей. Тип `type ('short' | 'empty')` обусловлен особенностями API и позволяет эмулировать получение пустых данных. Карточка новости (`NewsCard`) также конфигурируется через пропсы, поддерживает разные варианты отображения (`variant`) и возможность выделения главной новости (`isPrimary`)

#### Архитектура

Проект организован по методологии `Feature-Sliced Design`

#### Работа с данными

Для запросов к API используется связка `React Query` + `Axios`. `React Query` управляет состояниями загрузки, ошибки и успеха, а также кэширует данные, что исключает ручное управление флагами `isLoading/isError`

#### Адаптивность

Сетка карточек построена на `flexbox`, `media queries` и `container queries`, что обеспечивает корректное отображение как на десктопе, так и на мобильных устройствах

### Какие дополнительные требования удалось реализовать

- **Кэширование**: Использование `React Query` с настроенными параметрами кэширования позволило избежать повторных запросов уже загруженных страниц

- **Ленивая загрузка изображений**: Реализован хук `useLazyLoad`, который через `Intersection Observer API` откладывает загрузку изображения до его появления во вьюпорте

- **Анимации**: Для появления карточек новостей использована библиотека `Motion` (бывший Framer Motion)

- **Типизация**: Все компоненты и пропсы полностью типизированы с помощью TypeScript
